个性化推荐系统排序模型¶

本章主要介绍如何开发DeepFM模型实现推荐系统排序功能。推荐使用GPU环境运行相关程序，可以在启动环境时，如下图所示选择“高级版”环境即可。

如果没有算力卡资源可以点击链接申请。

概述¶

如果用户在访问商务网站或使用搜索引擎时，无法在第一时间获取自己感兴趣的内容，则会降低对网站或搜索引擎的评价，缩短使用时间或者使用频率，甚至放弃使用该网站或搜索引擎。因此目前大部分电子商务系统、社交网络和搜索引擎中，都不同程度的使用了各种形式的个性化推荐技术，帮助用户快速获取想要的信息。

个性化推荐系统一般包括两个模块，召回和排序。排序是指经过推理将用户感兴趣的内容按照点击率由高到低推荐给用户，在个性化推荐系统中决定了最终的推荐效果。对于数据量极大的业务，排序需要和召回模块配合使用，召回用来在全量的商品库中从不同的角度（标签匹配、深度模型、热门等）筛选出用户可能感兴趣的商品作为候选数据集，然后再由排序模块对候选集进行精准排序，推荐给用户。但是对于数据量不大的业务只使用排序模块即可。如果用户希望自己构建排序系统，实现个性化推荐功能，则可以使用飞桨的DeepFM模型。

说明：
如果用户想了解召回模型的相关内容，可以参考前一章召回教程。

飞桨的DeepFM推荐系统排序模型可以分析用户的反馈行为，包括点击、收藏、购买等，最为常见的是点击率(CTR)。通过分析这些数据找到用户个性化需求，为用户推荐其感兴趣的内容。本文以最常见的点击率（CTR）预估为例进行讲解。

基本原理¶

DeepFM模型的核心思想是使用特征组合来推理用户对物品或网页等内容的是否感兴趣的概率，不同的特征的组合对最终预测用户是否感兴趣有不同的贡献。一个商品的特征包括物品属性特征和用户属性特征等，这些特征从数据类型角度可以分为整数型特征和离散型特征，不同的特征之间可以相互进行组合。模型会根据组合成的特征来进行推理，计算出最终用户对目标物品或网页等内容是否感兴趣进行概率预测。

DeepFM模型的训练与二分类问题较为相似，训练使用的数据集是由样本特征和其对应的样本标签组成的，样本标签取值为0或1，1表示用户点击了该条内容，即表示用户对该条内容感兴趣，0表示未点击。训练过程中将标签作为预测目标。本节将重点讲述如何使用DeepFM成功训练模型，使用模型根据样本特征预测用户对样本内容感兴趣的程度。

DeepFM模型的网络结构如图1所示，从模型的名字上就可以看出来，该模型最主要的两个部分就是DNN Layer和FM Layer（Factorization Machines, 因子分解机）：

  • DNN Layer：随着深度学习的推广，DNN（Deep Neural Networks）、FNN（Factorization Machine supported Neural Network）、PNN（Probabilistic Neural Network）都对高阶特征有了更好的刻画，但是对低阶特征的刻画不甚理想。
  • FM Layer：FM Layer通过隐向量（latent vector）做内积来表示组合特征，但是实际应用中受限于计算复杂度，一般也就只考虑到二阶交叉特征，可以对低阶特征有很好的表示，但是缺乏高阶特征的表示。

为了同时考虑低阶和高阶特征，减少人工的特征工程，所以才提出了DeepFM模型，同时兼顾低阶特征和高阶特征。


图1 DeepFM模型结构图

DeepFM模型的排序过程包含以下几个部分：

  1. 数据预处理。该过程的目的是统计原始输入文件中各种特征的取值频率，构建特征字典。用户需要编写或使用已有脚本对文本进行处理，一般包括数据文件切分，训练/评估数据切分，构建特征字典等步骤，具体请参见下面“具体实现”中的“数据预处理”步骤。
  2. 预测用户对样本兴趣程度。训练样本在经过One-Hot编码成稀疏特征向量后输入到模型中，模型会将样本通过Dense Embeddings（嵌入层）处理后，生成稠密特征向量。模型中的FM Layer构造会使用稀疏特征向量和稠密特征向量生成二阶特征组合，深度神经网络(DNN)会使用稠密特征向量构造高阶特征组合，然后DeepFM模型会同时使用二阶特征组合和高阶特征组合来生成预估值来判断用户对一条样本内容是否感兴趣。
  3. 最后预估值和真实Label值经过log_loss函数计算得出Loss，模型将根据Loss值反向更新模型参数。

DeepFM的具体实现原理可以参考DeepFM: A Factorization-Machine based Neural Network for CTR Prediction。

DeepFM模型库主要包括如下文件：

  ├── README.md                       # 文档
  ├── train.py                        # 本地训练脚本
  ├── infer.py                        # 本地预测脚本
  ├── network.py                      # 网络结构
  ├── data_reader.py                  # 读取数据相关的函数
  ├── utility.py                      # 参数设置和通用函数
  ├── data/
      ├── download_preprocess.py      # 下载数据集并进行预处理的脚本
      ├── preprocess.py               # 数据预处理脚本

快速运行¶

如果用户希望使用经典数据集Criteo和DeepFM自带脚本快速训练出可以使用的模型，则可以按照如下步骤进行操作。

1. 数据集预处理¶

请下载并解压Criteo数据集到当前工作目录。解压后可以得到readme.txt，test.txt，train.txt三个文件。其中train.txt是带有标签的样本，可用来训练和评估模型。test.txt是没有标签的样本，在本示例中不使用。readme.txt是对数据集的说明。

如果用户在AI Studio下使用，可以在创建项目时添加Criteo数据集。若项目已经创建，可以在项目启动界面点击修改，然后点击添加数据集添加Criteo数据集。添加结束后重启环境，在data目录下可以找到dac.tar.gz文件。

如果用户使用的是Linux操作系统，并且安装了wget下载工具和tar压缩打包工具，则可以使用如下命令完成上述操作。

Criteo数据集是Kaggle一次广告点击比赛的官方数据集，广泛应用于论文的实验和比赛中。包含4500w用户的点击记录。每一行是一条记录，第一列是是否被点击，剩下的是用户和广告特征。

在预处理中我们按照min-max normalize的方法将连续性特征归一到[0,1]，同时过滤掉小于出现10次的类别特征，然后整个数据集被随机切分成90%的训练集和10%的测试集。

由于数据太大，我们已经提前下载好数据，如果想下载原始数据，请执行如下命令

wget https://s3-eu-west-1.amazonaws.com/kaggle-display-advertising-challenge-dataset/dac.tar.gz --no-check-certificate

预处理的脚本具体请参考https://github.com/PaddlePaddle/models/tree/develop/PaddleRec/ctr/deepfm_dygraph 。我们提前处理好数据放到了data目录下面

预处理完成后，可以在data目录中看到新生成了如下目录和文件。

在开始训练之前，用户需要通过数据预处理过程将训练使用的数据集进行转化，具体过程如下：
（1）数据文件切分：首先将数据文件分成固定大小的若干个小文件。切分的目的是方便打乱文件顺序，划分训练/预测数据集。
（2）训练/预测数据切分：按比例（默认为9:1）选取其中的部分小文件作为训练数据文件，其余的作为预测数据文件。
（3）构建特征字典：统计原始数据文件中各种特征的取值的频率，可以设定阈值来过滤出现频率小于阈值的特征取值，最后构建特征字典，将特征取值映射成整型id。

3. 定义模型¶

DeepFM的模型结构如图1所示，主要包括如下几个部分：

（1）稀疏特征层(Sparse Features)：稀疏特征输入层。推理使用的原始输入中包含13个整数型特征（例如： 年龄）以及26个离散型特征（例如： 性别）。 离散型数据需要进行one-hot编码，连续型数据可以先离散化再one-hot编码。 但是在经过one-hot编码之后，数据会维度会升高并且变得稀疏，即大部分值都为0，只有极少部分为1。

（2）嵌入层(Dense Embeddings)：将稀疏特征转换为稠密向量。由于CTR任务的输入特征十分稀疏，仅使用稀疏特征作为输入会导致网络参数过多，对学习效果产生影响，因此需要引入一个嵌入层将输入稀疏向量压缩到低维稠密向量，而且每个特征在Embedding后长度均相同。在之后的因子分解机和深度神经网络部分，这一层是共享的。
如下图所示，稠密向量$e^i$的计算公式如下所示。$X_{ij}$表示稀疏特征输入，$V$表示权重矩阵，即稀疏特征输入与稠密特征相连的权重$v_{ij}$组成的矩阵。 $$e^i=V^T{\cdot}X_{ij}$$ 以$e^1$为例，$V$对应的取值如下所示。由于图中Field1的稀疏层输入仅第二个值为1，这相当于选中了$V$中的第二行。 由此可见对于每一个Field在经过Embedding后都会选出中$V$中的某一行，将这些行拼接起来就得到了Embedding后的稠密特征。 $$V=\left[ \begin{matrix} v_{11} & v_{12} & v_{13} & v_{14} & v_{15}\\ v_{21} & v_{22} & v_{23} & v_{24} & v_{25} \\ v_{31} & v_{32} & v_{33} & v_{34} & v_{35} \\ v_{41} & v_{42} & v_{43} & v_{44} & v_{45} \\ v_{51} & v_{52} & v_{53} & v_{54} & v_{55} \end{matrix} \right]$$

（3）因子分解机(FM)：包含多个特征组合的内积计算，其功能是对低阶特征进行提取。通过对每一维特征的隐变量内积来提取特征组合，考虑到复杂度问题，在DeepFM中只用到了二阶特征组合。

公式如下所示：

  • $<w,X>$为一阶特征，其中$x$是稀疏层的标签值，$w$是图2中黑色连线的权重。
  • ${\sum_{j_1=1}^d}{\sum_{j_2=j_1=1}^d}<v_i,v_j>X_{j1}{\cdot}X_{j2}$为二阶特征组合，即低维稠密向量的两两组合，其中d是输入的特征维度，也就是Field的数量，$v_i$和$v_j$是嵌入层输出的稠密向量。其等价于和的平方减去平方和的一半，具体实现参考代码。 $$Y_{FM}=<w,X>+{\sum_{j_1=1}^d}{\sum_{j_2=j_1=1}^d}<v_i,v_j>X_{j1}{\cdot}X_{j2}={\sum_{i=1}^d}w_iX_i+ \frac{1}{2}{\sum_{f=1}^d}[({\sum_{i=1}^dv_{if}X_i})({\sum_{j=1}^d}v_{jf}X_j)-({\sum_{i=1}^d}v_{if}^2X_i^2)]$$

（4）深度神经网络(DNN)：包含多个全连接层，其功能是提取高阶特征$Y_{DNN}$。其输入是嵌入层的稠密向量的拼接。

（5）输出层(Output)：输出对样本的预测。将因子分解机和深度神经网络的输出相加，通过sigmoid激活函数得到最后的输出，其公式如下所示。 $$Y=\sigma(Y_{FM}+Y_{DNN})$$

模型的具体源码请参见deepfm/network.py。

DeepFM动态图的实现包括三个Layer，作为主Layer的DeepFM Layer会调用了FM Layer和DNN Layer。

  1. DeepFM Layer首先在init函数中定义了超参数，然后引入了两个Layer。
  2. 在Forward前向计算函数中根据输入调整数据格式，分别计算FM和DNN的结果，最终经过输出层得到预测值。

说明：
网络结构图中的Dense Embeddings层的计算是在FM Layer中实现的，该部分输出的稠密特征向量也会传入到DNN Layer中，即实现了FM Layer和DNN Layer的Embedding参数共享。

import math

import paddle.fluid as fluid
from paddle.fluid.dygraph.nn import Linear, Embedding


class DeepFM(fluid.dygraph.Layer):
    def __init__(self, args):
        super(DeepFM, self).__init__()
        self.args = args
        # 初始化值
        self.init_value_ = 0.1
        # FM Layer
        self.fm = FM(args)
        # DNN Layer
        self.dnn = DNN(args)

    def forward(self, raw_feat_idx, raw_feat_value, label):
        # reshape变量
        feat_idx = fluid.layers.reshape(raw_feat_idx,
                                        [-1, 1])  # (None * num_field) * 1
        feat_value = fluid.layers.reshape(
            raw_feat_value,
            [-1, self.args.num_field, 1])  # None * num_field * 1
        # 前向计算FM
        y_first_order, y_second_order, feat_embeddings = self.fm(feat_idx,
                                                                 feat_value)
        # 前向计算DNN
        y_dnn = self.dnn(feat_embeddings)
        # 综合FM和DNN 计算整体的预测值
        predict = fluid.layers.sigmoid(y_first_order + y_second_order + y_dnn)

        return predict

定义因子分解机FM Layer。FM Layer内部会实现了Dense Embeddings和FM的核心计算，其中Dense Embeddings的计算过程会生成稠密特征向量，稠密特征向量会和稀疏特征向量一起继续作为FM的计算的输入，计算出FM Layer的预估值。而且其中的稠密特征向量还会用做DNN Layer的输入，这样DNN和FM Layer就实现了共享dense embedding参数。

class FM(fluid.dygraph.Layer):
    # 在__init__()函数中创建了需要训练的参数。
    def __init__(self, args):
        super(FM, self).__init__()
        self.args = args
        self.init_value_ = 0.1
        # 创建嵌入层的参数
        # init_value表示参数初始化的值
        self.embedding_w = Embedding(
            size=[self.args.num_feat + 1, 1],
            dtype='float32',
            padding_idx=0,
            param_attr=fluid.ParamAttr(
                initializer=fluid.initializer.TruncatedNormalInitializer(
                    loc=0.0, scale=self.init_value_),
                regularizer=fluid.regularizer.L1DecayRegularizer(
                    self.args.reg)))
        self.embedding = Embedding(
            size=[self.args.num_feat + 1, self.args.embedding_size],
            dtype='float32',
            padding_idx=0,
            param_attr=fluid.ParamAttr(
                initializer=fluid.initializer.TruncatedNormalInitializer(
                    loc=0.0,
                    scale=self.init_value_ /
                    math.sqrt(float(self.args.embedding_size)))))
    # Forward函数中描述了前向计算的逻辑。
    def forward(self, feat_idx, feat_value):
        # -------------------- first order term  --------------------
        # 一阶特征
        first_weights_re = self.embedding_w(feat_idx)
        first_weights = fluid.layers.reshape(
            first_weights_re,
            shape=[-1, self.args.num_field, 1])  # None * num_field * 1
        #  w*x
        y_first_order = fluid.layers.reduce_sum(first_weights * feat_value, 1)

        # -------------------- second order term  --------------------
        feat_embeddings_re = self.embedding(feat_idx)
        feat_embeddings = fluid.layers.reshape(
            feat_embeddings_re,
            shape=[-1, self.args.num_field, self.args.embedding_size
                   ])  # None * num_field * embedding_size
        feat_embeddings = feat_embeddings * feat_value  # None * num_field * embedding_size

        # sum_square part
        # 和的平方
        summed_features_emb = fluid.layers.reduce_sum(
            feat_embeddings, 1)  # None * embedding_size
        summed_features_emb_square = fluid.layers.square(
            summed_features_emb)  # None * embedding_size

        # square_sum part
        # 平方和
        squared_features_emb = fluid.layers.square(
            feat_embeddings)  # None * num_field * embedding_size
        squared_sum_features_emb = fluid.layers.reduce_sum(
            squared_features_emb, 1)  # None * embedding_size
        # 和的平方减平方和的一半等价于二阶组合特征
        y_second_order = 0.5 * fluid.layers.reduce_sum(
            summed_features_emb_square - squared_sum_features_emb,
            1,
            keep_dim=True)  # None * 1
        # 由于DNN和FM共享嵌入层参数，所以返回嵌入层作为DNNLayer的输入
        return y_first_order, y_second_order, feat_embeddings

理论上DNN Layer是由多个全连接层组成的，在代码中是使用循环的机制来实现的，即在循环内定义了一个全连接层，然后通过循环是数据可以多次经过全连接层的处理。

class DNN(fluid.dygraph.Layer):
    def __init__(self, args):
        super(DNN, self).__init__()
        self.args = args
        self.init_value_ = 0.1
        # 创建参数
        sizes = [self.args.num_field * self.args.embedding_size] + self.args.layer_sizes + [1]
        acts = [self.args.act
                for _ in range(len(self.args.layer_sizes))] + [None]
        w_scales = [
            self.init_value_ / math.sqrt(float(10))
            for _ in range(len(self.args.layer_sizes))
        ] + [self.init_value_]
        self.linears = []
        # 循环全连接层的个数，创建参数
        for i in range(len(self.args.layer_sizes) + 1):
            linear = Linear(
                sizes[i],
                sizes[i + 1],
                act=acts[i],
                param_attr=fluid.ParamAttr(
                    initializer=fluid.initializer.TruncatedNormalInitializer(
                        loc=0.0, scale=w_scales[i])),
                bias_attr=fluid.ParamAttr(
                    initializer=fluid.initializer.TruncatedNormalInitializer(
                        loc=0.0, scale=self.init_value_)))
            self.add_sublayer('linear_%d' % i, linear)
            self.linears.append(linear)

    def forward(self, feat_embeddings):
        # 输入的embedding是FM层创建的共享参数
        y_dnn = fluid.layers.reshape(
            feat_embeddings,
            [-1, self.args.num_field * self.args.embedding_size])
        for linear in self.linears:
            y_dnn = linear(y_dnn)
        return y_dnn

开始训练循环。使用两层嵌套循环方式进行训练：

  • 内层循环负责整个数据集的一次遍历，遍历数据集采用分批次（batch）方式，每批次数据的处理过程如下：
    1. 数据经过模型的前向计算函数处理，计算出预测结果；
    2. 以预测结果和真实结果作为输入，通过损失函数计算损失函数值（loss）；
    3. 定期保存训练中的模型；
    4. 使用梯度反向传播函数更新模型参数，进入下一次内层循环。
  • 外层循环定义遍历数据集的次数，通过超参数中的nepochs进行设置。

注意：
必须把这部分代码写在dygraph.guard的范围之内。

        # 评价模型效果函数
        deepfm.eval()   # 切换模型至预测模式
        print("start eval model.")

            raw_feat_idx, raw_feat_value, label = zip(*data)
            raw_feat_idx = np.array(raw_feat_idx, dtype=np.int64)
            raw_feat_value = np.array(raw_feat_value, dtype=np.float32)

            predict = deepfm(raw_feat_idx, raw_feat_value, label)

            # 计算更新预测auc
            predict_2d = fluid.layers.concat([1 - predict, predict], 1)
            auc_metric_test.update(preds=predict_2d.numpy(), labels=label.numpy())  # 使用每个batch结果，更新auc值

        print("test auc is %.6f" % (auc_metric_test.eval()))

    # 加载模型
    start_epoch = 0
    if args.checkpoint:
        model_dict, optimizer_dict = fluid.dygraph.load_dygraph(args.checkpoint)
        deepfm.set_dict(model_dict)     # 加载模型参数
        optimizer.set_dict(optimizer_dict)  # 加载优化器参数
        start_epoch = int(os.path.basename(args.checkpoint).split("_")[-1]) + 1  # 解析下一个训练epoch
        print("load model {} finished.".format(args.checkpoint))
    
    # 开始训练
    for epoch in range(start_epoch, args.num_epoch):
        batch_id = 0    # 训练批次
        total_loss = 0.0    # epoch内loss累计
        auc_metric = fluid.metrics.Auc("ROC")   # 统计训练auc

        for data in train_reader():
            raw_feat_idx, raw_feat_value, label = zip(*data)
            raw_feat_idx = np.array(raw_feat_idx, dtype=np.int64)
            raw_feat_value = np.array(raw_feat_value, dtype=np.float32)
            label = np.array(label, dtype=np.int64)
            # raw_feat_idx: 样本特征索引
            # raw_feat_value: 样本特征值
            # label: 样本标签
            # 详细格式参考data_reader.py
            raw_feat_idx, raw_feat_value, label = [to_variable(i) for i in [raw_feat_idx, raw_feat_value, label]]

            # 获取模型前向传播结果
            # predict：对样本点击率的预测
            predict = deepfm(raw_feat_idx, raw_feat_value, label)
            
            # 损失函数计算
            loss = fluid.layers.log_loss(input=predict, label=fluid.layers.cast(label, dtype="float32"))
            batch_loss = fluid.layers.reduce_sum(loss)

            # epoch内loss累计
            total_loss += batch_loss.numpy().item()

            # 反向传播计算梯度
            batch_loss.backward()
            # 更新参数
            optimizer.minimize(batch_loss)
            # 清空梯度
            deepfm.clear_gradients()

            # 计算更新训练auc
            predict_2d = fluid.layers.concat([1 - predict, predict], 1)
            auc_metric.update(preds=predict_2d.numpy(), labels=label.numpy())

            # 定期打印训练auc
            if batch_id > 0 and batch_id % 100 == 0:
                print("epoch: {}, batch_id: {}, loss {:.6f}, auc {:.6f}".format(epoch, batch_id, total_loss / args.batch_size / 100, auc_metric.eval()))

                total_loss = 0.0    # 每次打印后重新累计loss

            batch_id += 1
        # 保存模型参数和优化器参数
        print("going to save epoch {} model and optimizer.".format(epoch))
        fluid.dygraph.save_dygraph(deepfm.state_dict(), model_path=os.path.join(args.model_output_dir, "epoch_" + str(epoch)))
        fluid.dygraph.save_dygraph(optimizer.state_dict(), model_path=os.path.join(args.model_output_dir, "epoch_" + str(epoch)))
        print("save epoch {} finished.".format(epoch))
        # 每个epoch结束后评估模型
        deepfm.eval()
        eval()
        # 切换模型至训练模式
        deepfm.train()
    print('finish.')
