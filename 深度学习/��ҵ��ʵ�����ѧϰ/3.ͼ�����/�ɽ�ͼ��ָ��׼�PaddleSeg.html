飞桨图像分割套件PaddleSeg¶

本章主要介绍如何使用PaddleSeg训练和部署图像分割模型。建议使用GPU环境运行相关程序，可以在启动环境时，如下图所示选择“高级版”环境即可。

概述¶

随着人工智能的发展，图像分割技术已经在交通控制、医疗影像和人脸识别等多个领域获得了广泛的应用。如果用户直接使用Python设计、编写图像分割模型并进行训练，则需要消耗较大的工作量。但是现在告诉您，只需要10行左右的代码和命令就可以完成不同应用领域的图像分割功能，你相信吗？这是真实的！如下图所示，只需要使用飞桨的PaddleSeg图像分割模型库即可这样快速使用图像分割技术。

飞桨的PaddleSeg语义分割库如同一个工具箱一样，覆盖了DeepLabv3+、U-Net、ICNet、PSPNet和HRNet五类主流的分割模型，并提供了多个损失函数和多种数据增强方法等高级功能，用户可以根据使用场景从PaddleSeg中选择出合适的图像分割方案，从而更快捷高效地完成从训练到部署的全流程图像分割应用。 PaddleSeg具备模块化设计、丰富的数据增强、高性能、工业级部署的特点：

  • 模块化设计：支持U-Net、DeepLabv3+、ICNet、PSPNet和HRNet五种主流分割网络，结合预训练模型和可调节的骨干网络，可以满足不同性能和精度的要求；PaddleSeg提供了不同的损失函数，如Dice Loss、BCE Loss等类型，通过选择合适的损失函数，可以强化小目标和不均衡样本场景下的分割精度。
  • 丰富的数据增强：基于百度视觉技术部的实际业务经验，内置10+种数据增强策略，可结合实际业务场景进行定制组合，提升模型泛化能力和鲁棒性。
  • 高性能：PaddleSeg支持多进程IO、多卡并行、跨卡Batch Norm同步等训练加速策略，结合飞桨核心框架的显存优化功能，可以大幅度减少分割模型的显存开销，更快完成分割模型训练。
  • 工业级部署：全面提供服务端和移动端的工业级部署能力，依托飞桨高性能推理引擎和高性能图像处理，开发者可以轻松完成高性能的分割模型部署和集成。并且通过Paddle-Lite，用户可以在移动设备或者嵌入式设备上完成轻量级、高性能的人像分割模型部署。

下面将分为基本原理、前置条件、快速运行和具体实现过程几个部分进行介绍：

  1. 基本原理：简单介绍DeepLabv3+、U-Net、ICNet、PSPNet和HRNet五类主流的分割模型的实现原理。
  2. 前置条件：实现图像分割模型训练与推理所需要的环境条件，例如需要安装的Python和飞桨的版本号、如何下载PaddleSeg代码库等内容。
  3. 快速运行：以眼底医疗分割数据集和U-Net模型为例指导用户快速完成模型训练并推理操作。
  4. 具体实现过程：详细讲解如何准备数据集和预处理，包括如何处理自备数据集，然后讲解如何下载预训练模型、如何设置配置文件确定数据增强方式以及选择损失函数、如何训练、评估和部署模型等流程。

基本原理¶

当前PaddleSeg主要包括DeepLabv3+、U-Net、PSPNet、ICNet和HRNet五种主流的图像分割模型。

DeepLabv3+¶

DeepLabv3+是DeepLab系列的最后一篇文章，其前作有DeepLabv1、DeepLabv2和DeepLabv3。在最新作中，作者结合编码器-解码器(encoder-decoder)结构和空间金字塔池化模块(Spatial Pyramid Pooling, SPP)的优点提出新的语义分割网络DeepLabv3+，在 PASCAL VOC 2012和Cityscapes数据集上取得新的state-of-art performance. 
其整体结构如下所示，Encoder的主体是带有空洞卷积(Atrous Convolution)的骨干网络，骨干网络可采用ResNet等常用的分类网络，作者使用了改进的Xception模型作为骨干网络。紧跟其后的空洞空间金字塔池化模块(Atrous Spatial Pyramid Pooling, ASPP)则引入了多尺度信息。相比前作DeepLabv3，DeepLabv3+加入decoder模块，将浅层特征和深层特征进一步融合，优化分割效果，尤其是目标边缘的效果。此外，作者将深度可分离卷积(Depthwise Separable Convolution)应用到ASPP和Decoder模块，提高了语义分割的健壮性和运行速率。

在PaddleSeg当前实现中，支持两种分类骨干网络的切换：

  • MobileNetv2：适用于移动设备的快速网络，如果对分割性能要求较高，请使用此骨干网络。
  • Xception：DeepLabv3+原始实现的骨干网络，兼顾了精度和性能，适用于服务端部署。

具体原理细节请参考Encoder-Decoder with Atrous Separable Convolution for Semantic Image Segmentation。

U-Net¶

U-Net [1] 起源于医疗图像分割，具有参数少、计算快、应用性强的特点，对于一般场景适应度很高。U-Net最早于2015年提出，并在ISBI 2015 Cell Tracking Challenge取得了第一。经过发展，目前有多个变形和应用。
原始U-Net的结构是标准的编码器-解码器结构。如下图所示，左侧可视为一个编码器，右侧可视为一个解码器。编码器由四个子模块组成，每个子模块包含两个卷积层，每个子模块之后又通过max pool进行下采样。编码器整体呈现逐渐缩小的结构，不断减少池化层的空间维度，缩小特征图的分辨率，以捕获上下文信息。
解码器呈现与编码器对称的扩张结构，逐步修复分割对象的细节和空间维度，实现精准的定位。解码器同样也包含四个子模块，分辨率通过上采样操作依次增大，直到与输入图像的分辨率基本一致。
该网络还使用了跳跃连接，即解码器每上采样一次，就以拼接的方式将解码器和编码器中对应相同分辨率的特征图进行特征融合，帮助解码器更好地恢复目标的细节。由于网络整体结构类似于大写的英文字母U，故得名U-Net。具体原理细节请参考U-Net:Convolutional Networks for Biomedical Image Segmentation。

PSPNet¶

基于语义分割的场景解析是计算机视觉中一个基础话题。自动驾驶、机器人感知等潜在应用领域都十分看重场景解析。场景解析的难度与场景和标签的多样性密切相关。
Pyramid Scene Parsing Network（PSPNet）起源于场景解析(Scene Parsing)领域。如图3所示，普通FCN面向复杂场景出现三种误分割现象：

  • 关系不匹配。将船误分类成车，显然车一般不会出现在水面上。
  • 类别混淆。摩天大厦和建筑物这两个类别相近，误将摩天大厦分类成建筑物。
  • 类别不显著。枕头区域较小且纹理与床相近，误将枕头分类成床。

图2 普通FCN误分割现象

PSPNet的出发点是在算法中引入更多的上下文信息来解决上述问题。为了融合图像中不同区域的上下文信息，PSPNet使用了特殊设计的全局均值池化操作（Global Average Pooling）和特征融合构造金字塔池化模块 (Pyramid Pooling Module)。PSPNet最终获得了2016年ImageNet场景解析挑战赛的冠军，并在PASCAL VOC 2012和Cityscapes数据集上取得了当时的最佳效果。具体原理细节请参考Pyramid Scene Parsing Network。


图3 PSPNet结构图

ICNet¶

Image Cascade Network（ICNet) 是一个基于PSPNet的语义分割网络，设计目的是减少PSPNet推断阶段的耗时。ICNet主要用于图像实时语义分割，例如交通监控等领域。ICNet由三个不同分辨率的子网络组成，将输入图像变换为不同的分辨率，随后使用计算复杂度高的网络处理低分辨率输入，计算复杂度低的网络处理分辨率高的网络，通过这种方式在高分辨率图像的准确性和低复杂度网络的效率之间获得平衡。并在PSPNet的基础上引入级联特征融合单元（cascade feature fusion unit），实现速度快且质量高的分割模型。
具体原理细节请参考ICNet for Real-Time Semantic Segmentation on High-Resolution Images。


图4 ICNet结构图

HRNet¶

现有的大多数方法都是从高分辨率到低分辨率网络(high-to-low resolution network)产生的低分辨率表征中恢复高分辨率表征。但是HRNet（HighResolution Net）能够在整个过程中保持高分辨率表示。如图5所示，HRNet以高分辨率子网开始作为第一阶段，逐个添加高到低分辨率子网以形成更多阶段，并且并行连接多分辨率子网。在整个过程中反复交换并行多分辨率子网络中的信息来进行重复的多尺度融合。在像素级分类、区域级分类和图像级分类中，证明了这些方法的有效性。 HRNet具有两个特点：

  • 从高分辨率到低分辨率并行连接各子网络。
  • 反复交换跨分辨率子网络信息。

这两个特点使HRNet网络能够学习到更丰富的语义信息和细节信息。 HRNet在人体姿态估计、语义分割和目标检测领域都取得了显著的性能提升。具体原理细节请参考Deep High-Resolution Representation Learning for Visual Recognition。


图5 HRNet结构图

用户可以根据使用场景选择合适的模型：

  • 如果您是语义分割的初学者，则推荐使用U-Net和PSPNet模型。
  • 如果您希望可以以较快的速度完成训练和预测，则推荐使用DeepLabv3+（MobileNetv2）或ICNet。
  • 如果您希望获得最佳的综合性能，则推荐使用HRNet或DeppLabv3+。

3. 模型训练¶

请执行如下命令训练模型。

python PaddleSeg/pdseg/train.py ${FLAGS} ${OPTIONS}
FLAGS 默认值 用途
--cfg None 指定配置文件路径。对于眼底医疗分割数据集的配置文件保存在./configs文件夹中。
文件夹中保存了不同模型类型的配置文件，包括deeplabv3p_xception65_optic.yaml、hrnet_optic.yaml、icnet_optic.yaml、pspnet_optic.yaml和unet_optic.yaml。
--use_gpu False 指定使用GPU训练，如果未配置则使用CPU进行训练。
--use_mpio False 在Linux系统下训练，可以配置该参数使用多进程I/O，通过提升数据增强的处理速度进而大幅度提升GPU利用率。windows平台下不支持该功能。
--use_tb False 指定使用TensorBoard记录训练数据。
--log_steps 10 指定训练日志的打印周期（单位为step）。
--debug False 指定打印debug信息，IOU等指标涉及到混淆矩阵的计算，会降低训练速度。
--tb_log_dir None 指定TensorBoard的日志路径。
--do_eval None 指定保存模型时进行效果评估。

命令中的OPTIONS参数是训练相关的配置选项，常用的配置选项如下所示：

  • BASIC.BATCH_SIZE：批处理大小。
  • SOLVER.LR：学习率。
  • TRAIN.PRETRAINED_MODEL_DIR：预训练模型路径。
  • TEST.TEST_MODEL：测试模型的路径。
  • AUG.VAL_CROP_SIZE：验证时图像裁剪尺寸（宽，高）。
  • AUG.TRAIN_CROP_SIZE：训练时图像裁剪尺寸（宽，高）。
  • MODEL.MODEL_NAME：模型名称。
  • DATASET.DATA_DIR：数据集主目录。

其它OPTIONS参数选择可以参见评估配置选项。
对于视盘分割场景下的U-Net模型训练命令如下所示。：

In [ ]:
!export CUDA_VISIBLE_DEVICES=0  # 指定可用的物理卡号

                      --do_eval \

                      BATCH_SIZE 4 \

4. 模型评估¶

训练完成后，用户可以使用评估脚本eval.py来评估模型效果。假设训练过程中遍历数据集的次数（EPOCH）为10，保存模型的间隔为5，即每遍历5次数据集保存1次训练模型。因此一共会产生2个定期保存的模型，加上最终保存的final模型，一共有3个模型。建议用户选择最终保存的模型进行效果的评估：

python PaddleSeg/pdseg/eval.py ${FLAGS} ${OPTIONS}
FLAGS 默认值 用途
--cfg None 指定配置文件路径。
--use_gpu False 指定使用GPU评估，如果未配置则使用CPU进行评估。
--use_mpio False 在Linux系统下评估，可以配置该参数使用多进程I/O，通过提升数据增强的处理速度进而大幅度提升GPU利用率。windows平台下不支持该功能。

命令中的OPTIONS参数是训练相关的配置选项，具体配置取值可以参见训练配置选项。
对于视盘分割场景下的U-Net模型评估命令如下所示。：

In [ ]:
!python PaddleSeg/pdseg/eval.py --use_gpu \
                     --cfg PaddleSeg/configs/unet_optic.yaml \
                     TEST.TEST_MODEL PaddleSeg/saved_model/unet_optic/final

在图像分割领域中，评估模型质量主要是通过三个指标进行判断，准确率（acc）、平均交并比（Mean Intersection over Union，简称mIoU）、Kappa系数。

  • 准确率：指类别预测正确的像素占总像素的比例，准确率越高模型质量越好。
  • 平均交并比：对每个类别数据集单独进行推理计算，计算出的预测区域和实际区域交集除以预测区域和实际区域的并集，然后将所有类别得到的结果取平均。在本例中，正常情况下模型在验证集上的mIoU指标值会达到0.70以上，显示信息示例如下所示，倒数第四行的IoU=0.8717即为mIoU。
  • Kappa系数：一个用于一致性检验的指标，可以用于衡量分类的效果。kappa系数的计算是基于混淆矩阵的，取值为-1到1之间，通常大于0。其公式如下所示，$P_0$为分类器的准确率，$P_e$为随机分类器的准确率。Kappa系数越高模型质量越好。 $$Kappa= \frac{P_0-P_e}{1-P_e}$$

具体实现过程¶

1. 准备数据集¶

在训练图像分割模型之前，用户需要准备训练、验证和测试时使用的数据集。如图7所示，一般的分割库使用单通道灰度图作为标注图片，但是这种标注图片显示为全黑的效果，无法直接观察标注是否正确，模型测试过程中也无法直接判断图像分割的效果。
为了解决上述问题，PaddleSeg在支持灰度标注图的基础上，同时支持了伪彩色标注图。伪彩色标注图是在原来单通道图片基础上，注入调色板。在图片大小基本不变的基础上，显示出彩色的效果。因此用户既可以选择灰度标注图作为数据集，也可以选择伪彩色标注图作为数据集。


图7 标注图片比较

PddleSeg已支持2种标注工具对图片做伪彩色标注，LabelMe和精灵数据标注工具。这两种工具可以对图片进行伪彩色标注，具体使用方法请参考：

  • LabelMe标注教程
  • 精灵数据标注工具教程

如果用户使用的数据集是预先准备好的灰度标注图，且希望检查标注效果，则可以执行gray2pseudo_color.py脚本将灰度标注图转换为伪彩色标注图。gray2pseudo_color.py脚本有两种使用方式：

  • 如果用户希望将指定目录下的所有灰度标注图转换为伪彩色标注图，则可以执行下面的命令调用脚本进行操作。
    • dir_or_file为灰度标注图所在目录。
      • output_dir为伪彩色标注图的输出目录。

python PaddleSeg/pdseg/tools/gray2pseudo_color.py <dir_or_file> <output_dir>

  • 如果用户仅希望将指定数据集中的部分灰度标注图转换为伪彩色标注图，则可以执行下面的命令调用脚本进行操作。
    • dir_or_file为灰度标注图列表所在目录。
      • output_dir为伪彩色标注图的输出目录。
      • dataset directory为数据集所在根目录。
      • file list separator为文件列表分隔符。

python PaddleSeg/pdseg/tools/gray2pseudo_color.py <dir_or_file> <output_dir> --dataset_dir <dataset directory> --file_separator <file list separator>

该命令的功能是调用脚本将数据集所在根目录下的灰度标注图片，按照dir_or_file指定的灰度标注图列表进行伪彩色标注图转换，列表中的图片文件名间的间隔符需要和file list separator参数取值相同。

4. 设置配置文件¶

用户需要创建配置文件来设置模型类型、训练批次等信息，配置文件的格式为yaml格式。PaddleSeg自带的配置文件保存在./configs目录中，配置文件的命名中都带有模型类型、骨干网络和预训练数据集等信息，用户可以根据这些信息挑选合适的配置文件修改。如下所示为全量配置文件信息，其上的注释信息可以帮助用户了解配置文件中的字段含义。

配置文件中除了可以设置常用的模型和训练信息之外，用户还可以规划数据增强策略（AUG）以及挑选损失函数（LOSS）：

  • 数据增强是指对数据集中的图片进行缩放（Resize）、翻转、变换（Rich Crop）和随机裁剪（Random Crop）等处理。使用处理后的训练数据训练模型，可以大幅度提升模型的鲁棒性。用户可以根据实际的场景灵活组合数据增强策略，让整个分割模型应用泛化能力更强。数据增强的具体信息请参考PaddleSeg数据增强。
  • PaddleSeg支持softmax_loss、dice_loss和bce_loss三种损失函数。默认PaddleSeg使用softmax_loss，但是对于二分类图像分割任务，往往存在类别分布不均的情况，例如瑕疵检测，道路提取及病变区域提取等，如图9所示。此类任务一般背景图像所占比例远大于前景比例，此时使用dice_loss的效果比softmax_loss更好，这是因为dice_loss通过预测和标注的交集除以它们的总体像素进行计算，它将一个类别的所有像素作为一个整体进行考量，计算交集在总体中的占比，所以不受大量背景像素的影响，因此对于该场景用户可以在配置文件中将损失函数设置为dice_loss。
    注意：
    softmax_loss不能与dice_loss或bce_loss组合使用，dice_loss可以和bce_loss组合使用，例如LOSS:["dice_loss","bce_loss"]。

    图9 不均衡的二分类图像分割

检查项 含义 检查内容
DATASET.SEPARATOR check 列表分割符校验 判断训练、验证和测试列表文件中的分隔符设置是否正确。
dataset reading check 数据集读取校验 通过读取训练、验证和测试数据集中所有图片，判断yaml配置文件中这3项设置是否正确。

label format check 标注格式校验 检查标注图像是否为PNG格式。标注图像请使用PNG无损压缩格式的图片，若使用其他格式则可能影响精度。
label gray check 标注通道数校验 检查标注图的通道数。正确的标注图应该为单通道图像。
label class check 标注类别校验 检查实际标注类别是否和配置文件中参数DATASET.NUM_CLASSES和DATASET.IGNORE_INDEX匹配。
标注图像类别数值必须在[0-(DATASET.NUM_CLASSES-1)]范围内或者为DATASET.IGNORE_INDEX。 标注类别建议从0开始，否则可能影响精度。
label pixel statistics 标注像素统计 统计每种类别的像素总数和所占比例。
DATASET.IMAGE_TYPE check 图像格式校验 检查配置文件中图片类型参数DATASET.IMAGE_TYPE是否设置正确。
当数据集包含三通道图片时，DATASET.IMAGE_TYPE设置为rgb；当数据集全部为四通道图片时，DATASET.IMAGE_TYPE设置为rgba。
max image size statistics 图像最大尺寸统计 统计数据集中图片的最大高和最大宽。
shape check 图像与标注图尺寸一致性校验 验证图像尺寸和对应标注图尺寸是否一致。

如果用户希望使用多GPU以便更快速的训练出模型，则可以在调用train.py脚本时添加paddle.distributed.launch参数进行训练，具体配置示例如下所示：

                      TRAIN.PRETRAINED_MODEL_DIR PaddleSeg/pretrained_model/unet_bn_coco \
                      SOLVER.LR 5e-5

如果用户希望使用多GPU进行训练且GPU是Turning和Volta架构，则可以启用混合精度训练，Turning和Volta架构是NVIDIA最新的两种GPU微体系架构。 混合精度训练速度更快，在配置方面需要将MODEL.FP16设置为True。
说明：
Turning和Volta架构是NVIDIA最新的两种GPU微体系架构。
具体配置示例如下所示：

In [ ]:
!export CUDA_VISIBLE_DEVICES=0,1
!python -m paddle.distributed.launch PaddleSeg/pdseg/train.py --use_gpu \
                      --do_eval \
                      --cfg PaddleSeg/configs/unet_optic.yaml \
                      BATCH_SIZE 4 \
                      TRAIN.PRETRAINED_MODEL_DIR PaddleSeg/pretrained_model/unet_bn_coco \
                      SOLVER.LR 5e-5 \
                      MODEL.FP16 True

缺省情况下，系统会采用动态scale的方式，若用户希望使用静态scale的方式进一步加快训练速度，则可以通过MODEL.SCALE_LOSS设置，但是具体取值需要用户根据经验或实验进行设置。具体配置示例如下所示：

In [ ]:
!export CUDA_VISIBLE_DEVICES=0,1
!python -m paddle.distributed.launch PaddleSeg/pdseg/train.py --use_gpu \
                      --do_eval \
                      --cfg PaddleSeg/configs/unet_optic.yaml \
                      BATCH_SIZE 8 \
                      TRAIN.PRETRAINED_MODEL_DIR PaddleSeg/pretrained_model/unet_bn_coco \
                      SOLVER.LR 5e-5 \
                      MODEL.FP16 True \
                      MODEL.SCALE_LOSS 512.0

# YAML_FILE_PATH为yaml配置文件路径，FINAL_PATH为模型导出路径
  python PaddleSeg/pdseg/export_model.py --cfg ${YAML_FILE_PATH} TEST.TEST_MODEL ${FINAL_PATH}
In [ ]:
!python PaddleSeg/pdseg/export_model.py --cfg PaddleSeg/configs/unet_optic.yaml TEST.TEST_MODEL PaddleSeg/saved_model/unet_optic/final

预测模型会导出到freeze_model目录，用于C++预测的模型配置会导出到freeze_model/deploy.yaml下。

7. PaddleSeg预测部署¶

为了帮助用户将训练好的模型集成到自己的服务中，PaddleSeg提供三种预测部署方案：

  • Python预测部署方案
  • C++预测部署方案
  • Paddle-Lite的移动端部署方案

Python预测部署方案¶

Python预测部署方案相关的脚本和文件所在的目录为./deploy/python/，包括如下文件：

├── README.md   # Python预测部署方案说明文件
├── requirements.txt   # Python预测部署方案依赖的Python包
└── infer.py  # 完成分割模型的预测以及结果可视化的脚本

Python预测部署方案的实现过程如下：

（1）请在预测部署的设备上安装PaddlePaddle1.6.1版本，具体安装方法请参见快速安装。
注意:
如果硬件环境为Turning和Volta架构的GPU，且需要使用TensorRT支持的FP16半精度优化功能， 则本步骤需要自行安装TensorRT并编译PaddlePaddle, 安装方法请参考编译安装参考文档。
（2）安装Python依赖包。在当前目录下, 使用pip安装Python依赖包。

In [ ]:
!pip install -r PaddleSeg/requirements.txt

（2）安装OpenCV相关依赖库。预测脚本中需要使用OpenCV，所以需要安装OpenCV相关的动态链接库。

  • Ubuntu下安装相关链接库：
apt-get install -y libglib2.0-0 libsm6 libxext6 libxrender-dev
  • CentOS 下安装相关链接库：
yum install -y libXext libSM libXrender

（3）检查配置文件

模型导出的目录通常包括三个文件:

├── model #  模型文件  
├── params # 参数文件  
└── deploy.yaml # 配置文件，用于C++或Python预测部署的配置文件

deploy.yaml配置文件的主要字段及其含义如下，用户可以根据实际情况进行修改:

DEPLOY:
    # 是否使用GPU预测
    USE_GPU: 1
    # 模型和参数文件所在目录路径
    MODEL_PATH: "/root/projects/models/deeplabv3p_xception65_humanseg"
    # 模型文件名
    MODEL_FILENAME: "__model__"
    # 参数文件名
    PARAMS_FILENAME: "__params__"
    # 预测图片的的标准输入尺寸，输入尺寸不一致会做resize
    EVAL_CROP_SIZE: (513, 513)
    # 均值
    MEAN: [0.5, 0.5, 0.5]
    # 方差
    STD: [0.5, 0.5, 0.5]
    # 分类类型数
    NUM_CLASSES: 2
    # 图片通道数
    CHANNELS : 3
    # 预测模式，支持 NATIVE 和 ANALYSIS
    PREDICTOR_MODE: "ANALYSIS"
    # 每次预测的 batch_size
    BATCH_SIZE : 3

（4）执行预测程序
请在终端执行如下命令进行预测：

!python PaddleSeg/deploy/python/infer.py --conf=${conf_dir} --input_dir ${input_dir} --use_pr=False

参数说明如下:

  • conf_dir：模型的Yaml配置文件路径。
  • input_dir：需要预测的图片目录。
  • use_pr：是否使用优化模型，默认为False。PaddleSeg 0.3.0版导出的为优化模型, 此前版本导出的模型即为未优化版本。优化模型把图像的预处理以及后处理部分融入到模型网络中使用GPU 完成，相比原来CPU 中的处理提升了计算性能。

运行后系统会扫描input_dir目录下所有指定格式图片，并在input_dir目录下生成预测mask和可视化的结果，预测mask会保存为png格式的文件，其作用是为每个像素进行分类，例如对于二分类模型，mask里面每个byte就是一个分类值0或者1。例如，对于图片a.jpeg, 预测mask为a_jpeg.png文件，而可视化结果则保存为a_jpeg_result.png 。

C++预测部署方案¶

用户在使用C++预测部署方案之前，请先配置C++的编译环境，请根据系统选择对应的编译指南进行配置：

  • Linux 编译指南
  • Windows 使用 Visual Studio 2019 Community 编译指南
  • Windows 使用 Visual Studio 2015 编译指南

C++预测部署方案相关的脚本和文件所在的目录为./deploy/cpp/，包括如下文件：

├── demo.cpp # 演示加载模型、读入数据、完成预测任务的C++代码  
├── conf  
│   └── humanseg.yaml # 示例人像分割模型的配置文件  
├── images  
│   └── humanseg # 示例人像分割模型的测试图片目录  
├── tools  
│   └── visualize.py # 示例分割模型结果的可视化脚本  
├── docs  
│   ├── linux_build.md # Linux 编译指南  
│   ├── windows_vs2015_build.md # windows VS2015编译指南  
│   └── windows_vs2019_build.md # Windows VS2019编译指南  
├── utils # 一些基础公共函数  
├── preprocess # 数据预处理相关代码  
├── predictor # 模型加载和预测相关代码  
├── CMakeList.txt # cmake编译入口文件  
└── external-cmake # 依赖的外部项目cmake（目前仅有yaml-cpp）

PaddleSeg的C++预测部署方案自带一个人像分割模型示例用于测试，该模型对应的C++程序文件为demo.cpp。如果用户希望基于自己的业务特点，编写调用模型的C++程序，则可以参考此文件。

模型导出的目录通常包括三个文件:

├── model # 模型文件
├── params # 参数文件
└── deploy.yaml # 配置文件，用于C++或Python预测部署的配置文件
deploy.yaml配置文件的主要字段及其含义如下，用户可以根据实际情况进行修改:

 DEPLOY:
    # 是否使用GPU预测
    USE_GPU: 1
    # 模型和参数文件所在目录路径
    MODEL_PATH: "/root/projects/models/deeplabv3p_xception65_humanseg"
    # 模型文件名
    MODEL_FILENAME: "__model__"
    # 参数文件名
    PARAMS_FILENAME: "__params__"
    # 预测图片的的标准输入尺寸，输入尺寸不一致会做resize
    EVAL_CROP_SIZE: (513, 513)
    # 均值
    MEAN: [0.5, 0.5, 0.5]
    # 方差
    STD: [0.5, 0.5, 0.5]
    # 分类类型数
    NUM_CLASSES: 2
    # 图片通道数
    CHANNELS : 3
    # 预测模式，支持 NATIVE 和 ANALYSIS
    PREDICTOR_MODE: "ANALYSIS"
    # 每次预测的 batch_size
    BATCH_SIZE : 3

用户在根据实际情况修改配置文件后，即可以通过编译C++程序执行预测。以自带的人像分割模型示例为例，在终端中切换到生成的可执行文件所在目录后，执行如下命令进行预测，其中conf为模型配置的Yaml文件路径，input_dir为需要预测的图片目录。

  • Linux 系统中执行以下命令：
    ./demo --conf=/root/projects/PaddleSeg/deploy/cpp/conf/humanseg.yaml --input_dir=/root/projects/PaddleSeg/deploy/cpp/images/humanseg/
  • Windows 中执行以下命令： 
    D:\projects\PaddleSeg\deploy\cpp\build\Release>demo.exe --conf=D:\\projects\\PaddleSeg\\deploy\\cpp\\conf\\humanseg.yaml --input_dir=D:\\projects\\PaddleSeg\\deploy\\cpp\\images\humanseg\\

预测结果将会保存在input_dir目录中的xxxx_mask.png中。例如对demo.jpg图片进行预测，则预测结果文件为demo_jpg_mask.png。但是预测结果的png文件不能直接看到效果，必须经过可视化处理。对于二分类的图像分割模型。如果需要对预测结果进行可视化，请执行如下命令对预测结果进行可视化。

# 假设当前路径为PaddleSeg根目录
# 切换到脚本所在目录
cd ./deploy/cpp/tools/
# 拷贝保存分割预测结果的图片到本目录
cp XXX/demo_jpg.png
# 运行可视化脚本
python visualize.py demo.jpg demo_jpg_mask.png vis_result.png
